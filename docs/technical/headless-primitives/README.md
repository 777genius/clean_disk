# Headless Primitive RFC Index

This folder breaks the Headless primitive work into separate community-grade
RFCs. The goal is to design primitives that are useful for Clean Disk and also
good enough to become public Headless standards.

## Status

Design direction accepted. Not implemented yet.

For Clean Disk implementation priority, use
[Headless Clean Disk priority index](../headless-clean-disk-priority-index.md).
This folder is the broad public primitive catalog. The priority index decides
what Clean Disk builds first and what remains architecture-only.

## Global Architectural Rule

Headless primitives must follow existing Headless direction:

- component owns behavior, state machine, keyboard handling, and root
  accessibility;
- renderer owns visuals only;
- renderer receives commands, not application callbacks;
- app/product data stays outside Headless;
- public APIs use stable typed contracts, not stringly IDs;
- controlled/uncontrolled state follows Flutter POLA;
- component packages do not depend on other component packages;
- reusable mechanics go into `headless_foundation`;
- visual renderer contracts go into `headless_contracts`;
- Material/Cupertino implementations live in preset packages.

## Dependency Shape

```text
headless_foundation
  collection/
  grid/
  tree/
  viewport/

headless_contracts
  tree_grid/
  split_pane/
  context_menu/
  dialog/
  tooltip/
  status_region/

components/headless_tree_grid
components/headless_split_pane
components/headless_context_menu
components/headless_dialog
components/headless_tooltip
components/headless_status_region

headless_material
  implements renderer capabilities

packages/design_system
  wraps Headless for Clean Disk
```
